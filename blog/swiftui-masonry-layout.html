Building a Custom Masonry Layout in SwiftUI

February 12, 2026  ·  5 min read

The new Layout protocol in SwiftUI finally makes complex grid arrangements possible without reaching for UIKit. I built Spectral's wallpaper grid entirely with it — here's how, with annotated code.

Why Not LazyVGrid?

SwiftUI's built-in LazyVGrid with .flexible() columns gets you 80% of the way there. But it forces uniform row heights — every cell in a row is as tall as the tallest item. For a wallpaper app where images have different aspect ratios, this means either cropping everything to a square or leaving ugly whitespace gaps.

A masonry layout (like Pinterest's grid) places each item into the shortest column at the time of insertion, allowing variable heights while keeping columns balanced. Before SwiftUI's Layout protocol, you had to either use a UICollectionView compositional layout or manually calculate frames with GeometryReader hacks. Both approaches are verbose and fragile.

The Layout Protocol in 60 Seconds

The Layout protocol requires two methods:

  • sizeThatFits(proposal:subviews:cache:) — return the total size your layout needs
  • placeSubviews(in:proposal:subviews:cache:) — position each child view

SwiftUI calls these during layout passes. You get a subviews collection that lets you measure each child's ideal size before placing it. That's the key — you can ask "how tall would this image be if it were 180pt wide?" before you commit to placing it anywhere.

The MasonryLayout Implementation

struct MasonryLayout: Layout {
    var columns: Int
    var spacing: CGFloat

    func sizeThatFits(
        proposal: ProposedViewSize,
        subviews: Subviews,
        cache: inout Void
    ) -> CGSize {
        let columnWidth = columnWidth(total: proposal.width ?? 0)
        let heights = columnHeights(subviews: subviews, columnWidth: columnWidth)
        return CGSize(
            width: proposal.width ?? 0,
            height: heights.max() ?? 0
        )
    }

    func placeSubviews(
        in bounds: CGRect,
        proposal: ProposedViewSize,
        subviews: Subviews,
        cache: inout Void
    ) {
        let colWidth = columnWidth(total: bounds.width)
        var colHeights = Array(repeating: bounds.minY, count: columns)

        for subview in subviews {
            let col = colHeights.indices.min(by: { colHeights[$0] < colHeights[$1] })!
            let x = bounds.minX + CGFloat(col) * (colWidth + spacing)
            let size = subview.sizeThatFits(.init(width: colWidth, height: nil))
            subview.place(
                at: CGPoint(x: x, y: colHeights[col]),
                proposal: .init(width: colWidth, height: size.height)
            )
            colHeights[col] += size.height + spacing
        }
    }

    private func columnWidth(total: CGFloat) -> CGFloat {
        (total - spacing * CGFloat(columns - 1)) / CGFloat(columns)
    }

    private func columnHeights(subviews: Subviews, columnWidth: CGFloat) -> [CGFloat] {
        var heights = Array(repeating: CGFloat(0), count: columns)
        for subview in subviews {
            let col = heights.indices.min(by: { heights[$0] < heights[$1] })!
            let size = subview.sizeThatFits(.init(width: columnWidth, height: nil))
            heights[col] += size.height + spacing
        }
        return heights
    }
}

The core of it is the "shortest column first" strategy in placeSubviews: for each subview, find the column with the minimum accumulated height and place the item there. This keeps the columns balanced without needing to know image dimensions up front.

Using It in a View

MasonryLayout(columns: 2, spacing: 12) {
    ForEach(wallpapers) { wallpaper in
        WallpaperCard(wallpaper: wallpaper)
            .aspectRatio(wallpaper.aspectRatio, contentMode: .fit)
    }
}

Each WallpaperCard declares its own aspect ratio, and the layout engine figures out the height from the column width. No manual size calculations anywhere in the view layer.

Performance Considerations

The Layout protocol is not lazy — all subviews are measured before any are placed. For very large grids, wrap the layout in a ScrollView with LazyVStack chunking rather than passing thousands of subviews directly.

Spectral pages its wallpaper grid in batches of 40. Each batch is a separate MasonryLayout inside a LazyVStack, which gives you lazy loading at the batch level while keeping the masonry algorithm simple and stateless within each batch.

The Result

The final grid looks exactly like a native photo grid — variable heights, balanced columns, no cropping. And the implementation is under 60 lines of Swift with no UIKit bridging. The Layout protocol is one of the best additions to SwiftUI in recent years, and masonry is the perfect use case to demonstrate why.
